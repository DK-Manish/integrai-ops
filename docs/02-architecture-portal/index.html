05 · MCP Architecture

MCP exposes operational systems as safe, callable tools.

Agents can query approved tools for incident context without directly coupling the AI layer to every backend system.

MCP Server

Hosts approved tools and exposes a controlled tool interface.

Tool Registry

Defines what each tool can access, return and validate.

AI Agent

Calls tools during investigation to enrich reasoning.

Audit Layer

Records tool calls, inputs, outputs and investigation context.

06 · Event Processing

IntegrAI Ops treats operational changes as events so failures, retries and recovery actions can be replayed and explained.

07 · Data Model

Core entities are designed around incidents and integration timelines.

Incident

Status, severity, source, detected time, owner.

Integration

Service name, system boundary, API/workflow/event metadata.

Timeline Event

Timestamped operational facts linked to an incident.

AI Insight

Explanation, confidence, evidence and recommended actions.

Tool Call

MCP tool name, request context, response and audit result.

User

Role, permissions, team and notification preferences.
